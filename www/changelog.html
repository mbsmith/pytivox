Changes in 1.3

  • Fixes to make it run correctly under snow leopard.
  • Fix for pyTivo failing to pull .tivo files
  • StreamBaby will now push different episodes of the same show into a folder
  • many pytivo improvements (see pytivo (changes) ) including the ability to auto-resume transfers.

    Changes in 1.2.1

    Minor bugfix release.
    • pyTivo was attempting to use port 0, fixed
    • .tivo files can be pushed/pulled again

    Changes in 1.2

    • Latest version of:
      • pyTivo: changelog
      • streambaby: changelog
      • ffmpeg : Support many more video formats changelog
    • New preferences window. Still need to hit apply after changing preferences.
    • pyTivo and streambaby now support push of videos from the computer to any tivo in the house. You will need to provide your password for http://tivo.com/ in order to use it.
    • streambaby config: sort by filename or sort by title.
    • Support non unicode charsets in pytivo metadata
    • Enable more explicit logging to help debug future issues. Logs now under user/Library/Logs/pyTivoX
    • Streambaby image cache moved to user/Library/Cache/pyTivoX/
    • Streambaby can now parse more mp4 files to determine if they can be streamed instead of transcoded.
    • pyTivoX now ships as a dmg instead of a zip-file
    • Streambaby now presents itself as 'pyTivoX - [hostname]' in the photos,music,showcases list (must hit 'apply' for this to take effect)
    • bugfix: pytivo should no longer generate errors on very slow/old computers
    • bugfix: pytivo should serve music correctly now

    • swapped 'accept' and 'exit' buttons (bedelman)
